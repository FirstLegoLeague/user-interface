XY Grid Layout

The XY Grid Layout is the most abundant way to organize layouts in the system. Using it, you can create a large variaty of rectangular areas in the page.

There are two types of grids. A x-grid and a y-grid. A x-grid is divided horizontaly, providing control over the width of the columns, while a y-grid is divided vertically, providing control over the height of the rows.

Inside the Grid

Cells

The children elements of the grid are cells. Each grid is divided into 12 parts, and each cell can have an offset and size. In x-grid this division regards the horizontal layout, and in y-grid this division regards the vertial layout.

In order to create responsive grids, that match themselves to the size of the screen, foundation created classes based on the screen's size, with the three options: small, medium and large. So, for example, if I wanted a cell to be of size 3 and with offset 2 when the screen is small, My cell would look like this: <div class="cell small-3 small-offset-2"></div>. The classes follow the pattern [size]-[n], and [size]-offset-[n]. Notice that if there is only one class regarding the side of the cell, it would apply to all sizes.

Cell classes reference

  • cell - Declares this element to be a cell in the table
  • [size]-[n] - Defines the size [n] of this cell regarding the grid's axis
  • [size]-offset-[n] - Defines the offset [n] of this cell regarding the grid's axis
  • auto - Tells the cell to fill the ramining space
  • [size]-auto - Spcifies the auto class for a specific screen size

Outside the Grid

Grid container

A grid container is meant to control the size of the grid. It is 1200px wide by default, and has sides padding of 20-30px depending on the screen size.

Grid container classes reference

  • fluid - Removes the 1200px limit on the width of the container
  • full - Removes the width limit and the sides padding

Grid classes reference

  • grid-x - Declares this element to be the root of a x-grid
  • grid-y - Declares this element to be the root of a y-grid
  • grid-margin-x - Adds the grid margins on the sides of each cell
  • grid-margin-y - Adds the grid margins on the top and bottom of each cell
  • grid-padding-x - Adds the grid paddings on the sides
  • grid-padding-y - Adds the grid paddings on the top and bottom
  • [size]-up-[n] - Tells the table to display maximum [n] cells when in screen size [size], regarding the grid's direction
  • [size]-[margin|padding]-collapse - Tells the table to not have margin/padding in this specific screen size

Examples

Simple equal-width horizontal grid with four cells
<div class="grid-x"> <div class="cell auto"> <div class="cell auto"> <div class="cell auto"> <div class="cell auto"> </div>
Using [size]-[n] and [size]-offset-[n]
<div class="grid-x"> <div class="cell small-5"> <div class="cell small-4"> <div class="cell small-2 small-offset-1"> </div>
